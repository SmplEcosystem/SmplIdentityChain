About

The Smpl DID method specification conforms to the requirements specified in the DID specification currently published by the W3C Credentials Community Group. For more information about DIDs and DID method specifications, please see the DID Primer and DID Spec.

Smpl is a public ledger designed specifically for privacy-preserving self-sovereign identity.

 

Smpl DID Method

The namestring that shall identify this DID method is: smpl

A DID that uses this method MUST begin with the following prefix: did:smpl. Per the DID specification, this string MUST be in lowercase. The remainder of the DID, after the prefix, is the NSI specified below.

Target System(s)

Namespace Specific Identifier (NSI)

The Smpl DID scheme is defined by the following ABNF:

smplrin-did = "did:smpl:" idstring *(":" subnamespace)
idstring = 21*22(base58char)
subnamespace = ALPHA *(ALPHA / DIGIT / "_" / "-")
base58char = "1" / "2" / "3" / "4" / "5" / "6" / "7" / "8" / "9" / "A" / "B" / "C"
    / "D" / "E" / "F" / "G" / "H" / "J" / "K" / "L" / "M" / "N" / "P" / "Q"
    / "R" / "S" / "T" / "U" / "V" / "W" / "X" / "Y" / "Z" / "a" / "b" / "c"
    / "d" / "e" / "f" / "g" / "h" / "i" / "j" / "k" / "m" / "n" / "o" / "p"
    / "q" / "r" / "s" / "t" / "u" / "v" / "w" / "x" / "y" / "z"

Namestring Generation Method

The 128-bit uuid underlying a Smpl DID can be generated in various ways--using standard uuid methods.

A convenient regex to match smpl DIDs is:

^[1-9A-HJ-NP-Za-km-z]{21,22}$

A convenient regex to match the entire did string is:

^did:smpl:[1-9A-HJ-NP-Za-km-z]{21,22}(?<namespace>(?::\w[-\w]*)*)$

Examples

A valid smpl DID might be: did:smpl:2wJPyULfLLnYTEFYzByfUR.
